Reusable pipeline: GitHub -&gt; Build -&gt; Docker Build -&gt; Deploy via SSH.

<h3>Usage</h3>
<pre>
@Library('infra-shared-library') _

standardDeployPipeline(
    targetEnv: 'staging',
    targetHost: 'staging.example.com',
    imageName: 'portfolio-app'
)
</pre>

<h3>Parameters</h3>
<ul>
  <li><b>targetHost</b> (required) - host to deploy to</li>
  <li><b>imageName</b> (required) - Docker image name to build/tag</li>
  <li><b>targetEnv</b> - environment label for logging (default: staging)</li>
  <li><b>sshCredentialsId</b> - Jenkins credentials ID for the deploy SSH key (default: deploy-ssh-key)</li>
  <li><b>deployUser</b> - SSH user on the target host (default: deploy)</li>
  <li><b>remoteDir</b> - directory on the target host with the app's docker-compose.yml (default: /opt/app)</li>
  <li><b>buildStep</b> - closure with app-specific build steps (default: none)</li>
</ul>
